## Challenge
Create a user with a non-interactive shell on the server "App-server-3".

## Description
A non-interactive shell is used for users who need an account on the system but should not be able to log in interactively (e.g., service accounts, SFTP-only users). The `/usr/sbin/nologin` shell prevents interactive login while still allowing the account to exist for other purposes.

## Solution

### Step 1: SSH into the Server
Connect to the target server using SSH:

```bash
ssh user@server_ip
# Example:
ssh john@172.168.xx.xx
```

### Step 2: Create the User with Non-Interactive Shell
Create the user with the `-s` flag to specify the non-interactive shell:

```bash
sudo useradd -m -s /usr/sbin/nologin username
```

### Flags Explained
| Flag | Description |
|------|-------------|
| `-m` | Creates a home directory for the user |
| `-s /usr/sbin/nologin` | Sets the user's shell to nologin, preventing interactive login |

## Verification
To verify the user was created correctly:

```bash
grep username /etc/passwd
```

The output should show `/usr/sbin/nologin` as the shell.
